
DM Objective-Caml for Image Compression using Quad-trees : 

Structure du dossier:
    
    * Fichiers portant le nom :

        - DMFIle.ml : 
            Fichier contenant le code demandé dans le pdf du DM + quelques fonctions utilisées pour output les structures dans le terminal.
        
        - DMFile.mli :
            fichier contenant l'interface "passeport" du fichier DMFile.ml
        
        - main.ml :
            Fichier implémentant un main démontrant un cas possible d'utilisation d'un main () et du code du DM. 
        
        - file :
            Fichier compilé permettant de run le main ( ex commande : ./file info )
        
    * Dossier "outputs" :
        Vous pourrez y voir les images représentants les différents outputs des exos du DM (Qi <=> question i). 

    * Dossier rapport :
        Contient le .pdf structuré par LaTeX (contient tous les commentaires et explications)

Remarque: 
Toutes les explications/commentaires sont dans le rapport pour une raison de lisibilité (jupyter-notebook a été utilisé, ainsi que vscode).
De plus aucune explication explicite n'a été faite concernant les fonctions non-liées directement au DM, i.e toute la partie BONUS car aurait ajouté trop de "bruit" et
ne sont pas complexes à la lecture du code (transformations, cast etc).
